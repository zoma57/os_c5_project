TEST SCENARIOS DOCUMENTATION (C5 Project)
=========================================

1. Scenario A: Basic mixed workload
- Quantum: 3
- Input: P1(0,5), P2(1,3), P3(2,8), P4(3,6)
- Expected Behavior: The system should handle a mix of short and long bursts. SJF should preempt P1 when P2 arrives. Round Robin should time-slice equally.
- Actual Output: Matches expected. SJF provided lower average waiting time due to prioritizing P2, while RR ensured fair distribution of CPU time.

2. Scenario B: Short-job-heavy case
- Quantum: 2
- Input: P1(0,12), P2(1,2), P3(2,1), P4(3,2)
- Expected Behavior: SJF behavior should become very visible by immediately serving the short jobs (P2, P3, P4) and preempting the long job (P1). 
- Actual Output: Matches expected. SJF rapidly finished P2, P3, and P4, drastically lowering the average response and waiting times compared to RR, which forced short jobs to wait in the queue behind P1.

3. Scenario C: Fairness case
- Quantum: 4
- Input: P1(0,8), P2(0,8), P3(0,8)
- Expected Behavior: SJF will act like FCFS because all burst times are equal, causing P3 to wait a long time. RR should rotate perfectly, giving each process 4ms slices.
- Actual Output: Matches expected. RR demonstrated perfect fairness, giving all 3 processes an identical initial response time, whereas SJF made P3 wait 16ms before its first execution.

4. Scenario D: Long-job sensitivity case
- Quantum: 4
- Input: P1(0,15), P2(2,2), P3(3,3)
- Expected Behavior: P1 is a very long job. Under SJF, P1 will suffer as it gets preempted by P2 and P3. Under RR, P1 will get its fair share initially without being completely starved.
- Actual Output: Matches expected. SJF gave better overall efficiency, but RR proved better for the long job's first response, highlighting the fairness vs. efficiency trade-off.

5. Scenario E: Validation case
- Quantum: -1 (Invalid)
- Input: P1(-2, 5) [Invalid Arrival], P2(0, 0) [Invalid Burst], P3(0, 5) [Duplicate ID with another P3]
- Expected Behavior: The simulator must reject negative arrival times, zero/negative bursts, duplicate process IDs, and negative quantums safely.
- Actual Output: The system successfully blocked the invalid inputs and displayed the appropriate error messages ("Quantum must be > 0" and "Invalid Data") without crashing.
